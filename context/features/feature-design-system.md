# Feature Specification — Design System & Visual Language

## Purpose

The Design System defines:
- visual consistency
- interaction quality
- spatial rhythm
- UI behavior standards
- component philosophy
- product tone

The system should create a product that feels:
- calm
- premium
- operational
- focused
- modern
- AI-native
- strategically restrained

This is NOT:
> decorative UI design.

This IS:
> operational product craftsmanship.

---

# Design Philosophy

## Core Product Feeling

Foundary should feel like:

> "A calm operating system for high-velocity venture teams."

Inspired by:
- Linear
- Vercel
- Raycast
- modern AI-native SaaS products

The UI should communicate:
- operational confidence
- strategic clarity
- disciplined restraint
- execution momentum

---

# Design Principles

# 1. Calm High-Performance UX

The interface should feel:
- focused
- lightweight
- intentional
- stable

Avoid:
- visual chaos
- excessive decoration
- productivity gimmicks
- startup hype aesthetics

---

# 2. Density With Breathing Room

The UI should be:
- information-efficient
- compact
- highly scannable

BUT:
- never cramped
- never noisy
- never overwhelming

This balance is EXTREMELY important.

---

# 3. Restraint Creates Sophistication

Premium products often feel premium because:
- they avoid excess
- they reduce visual noise
- they preserve focus

Prefer:
- subtle hierarchy
- quiet confidence
- clean spacing
- restrained motion

Avoid:
- visual over-engineering
- aggressive styling
- loud interactions

---

# 4. Operational Readability

Every interface should optimize for:
- scanning
- prioritization
- workflow clarity
- decision-making speed

Users should quickly identify:
- importance
- risk
- status
- ownership
- momentum

---

# 5. Motion Should Support Clarity

Motion exists to:
- preserve continuity
- improve orientation
- reinforce flow

NOT:
- entertain users
- show off animation
- create distraction

---

## Product Aesthetic

- create visual overload

# Recommended Palette Direction

## Neutral Foundation

Use:
- deep charcoal backgrounds
- muted grays
- subtle surface layering

Avoid:

- saturated UI surfaces

# Accent Usage

Accent colors should:
- communicate meaning
- guide attention
- remain restrained

Accent colors should NOT:
- overpower interfaces
- create rainbow dashboards
- dominate cards

---

```txt
Urgent → Red
High → Orange
Medium → Blue
Low → Gray
```

---

## Status Colors

